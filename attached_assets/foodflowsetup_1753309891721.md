# Project Name: FoodFlow

## Description
FoodFlow is a comprehensive food delivery application that enables seamless interaction between customers, vendors, drivers, and admins, facilitating efficient order management and delivery tracking.

## Primary Goals
- Provide an intuitive platform for customers to explore menus, place orders, and track deliveries.
- Enable vendors to effectively manage their menus and order fulfillment.
- Equip drivers with tools for efficient delivery management and status updates.
- Allow admins to oversee operations, manage users, and generate insightful reports.

----------------------------------------------------

## Tech Stack & Environment

**Frontend:** Vite + React + TypeScript + Tailwind CSS

**Backend:** Supabase (PostgreSQL + Auth)

**Target Platform:** Single-Page Application

**Framework Versions & Config:** Latest stable versions

----------------------------------------------------

## Requirements & Features

### Core Features:
- **Customer Portal:** Browse menus, place orders, and track deliveries.
- **Vendor Dashboard:** Manage menus, receive orders, and update fulfillment status.
- **Driver Interface:** Access delivery assignments, navigation, and status updates.
- **Admin Panel:** Oversee system, manage users, and generate reports.

### User Flows
- **Customer Flow:** Sign up/log in, browse menus, select items, place order, track order status, receive delivery.
- **Vendor Flow:** Log in, update menu, receive orders, update order status, manage inventory.
- **Driver Flow:** Log in, view deliveries, navigate to locations, update delivery status.
- **Admin Flow:** Log in, manage users, monitor system, generate reports.

### Business Rules
- Customers can only place orders if logged in.
- Vendors can only update menus during predefined hours.
- Drivers must update status at each delivery stage.
- Admins have full access to all user data and reports.

## UI/UX Design

### Layout
- **Responsive Design:** Mobile-first approach ensuring seamless experience across devices.
- **Intuitive Navigation:** Clear and concise menus and buttons for easy accessibility.

### Look & Feel
- **Modern and Clean Design:** Utilize Tailwind CSS for a sleek, professional look.
- **Consistent Branding:** Uniform color schemes and fonts throughout the application.

### Pages
- **Home Page:** Introduction and call to action for new users.
- **User Dashboard:** Personalized experience based on user role (Customer, Vendor, Driver, Admin).
- **Order Page:** Detailed order and tracking information.
- **Admin Reports:** Visual reports and analytics.

### Components
- **Menu Card:** Displays food items with descriptions and prices.
- **Order Tracker:** Real-time order status display.
- **User Profile:** Editable user information.
- **Report Generator:** Admin tool for creating reports.

----------------------------------------------------

## Data Model & Supabase Setup

### Database Schema
- **Users Table:** Stores user information categorized by roles.
- **Orders Table:** Captures order details and statuses.
- **Menus Table:** Contains vendor menu information.
- **Deliveries Table:** Tracks driver assignments and status.

### Row-Level Security
Enabled by default

### Auth Requirements
- Email authentication for all user roles.
- Role-based access control to ensure appropriate access levels.
